# F22 Notes

- A F22 fecha a fase como release tecnica, nao como nova feature de produto.
- O quickstart publico continua local e `sync-first`; preview de artifacts aparece como capacidade adicional e opcional, nao como requisito da primeira run minima.
- A release notes deve deixar explicito que `RUN_REPORT.md` e previews dependem de artifacts persistidos ja existentes.
- Se a readiness revelar lacuna funcional real fora de docs/testes, o ajuste deve ser minimo e justificado; nada de abrir hardening amplo por inercia.
